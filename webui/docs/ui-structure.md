# Botical Web UI - Structure & Navigation Design

## Vision

Botical Web UI is a **mission-oriented development environment** - a lightweight IDE with AI agents at its core. Users work across multiple projects simultaneously via tabs, with AI-powered "missions" handling focused, autonomous work.

**Design Principles:**
1. **Mission-first**: Missions are the primary unit of work, not chat sessions
2. **Multi-project**: Work across projects via tabs, not a project switcher
3. **Process visibility**: Commands and services are first-class citizens
4. **Git-native**: Version control is integrated, not bolted on
5. **Keyboard-driven**: Power users can do everything without a mouse
6. **Dense but clear**: Show information efficiently without clutter

### Definitions

**Project**: Long-lived container with persistent file system, git history, and services.

**Mission**: Planned, goal-oriented work. Requires a planning document with completion criteria. The agent drafts the plan; the user approves before execution begins.

**Task**: Immediate work unit. No planning phase - execution starts right away. Lives within a project, optionally within a mission.

**Service**: Long-lived process (dev server, file watcher, etc.) bound to project/mission lifecycle.

**Command**: Short-lived process within a task. Runs and completes.

## Keyboard Shortcuts

### Global

| Shortcut | Action |
|----------|--------|
| `Cmd+K` | Command palette |
| `Cmd+P` | Quick open file |
| `Cmd+Shift+P` | Command palette (commands only) |
| `Cmd+B` | Toggle sidebar |
| `Cmd+J` | Toggle bottom panel |
| `Cmd+\` | Toggle terminal/process panel |
| `Cmd+,` | Open settings |
| `Cmd+Shift+N` | New window |

### Tabs

| Shortcut | Action |
|----------|--------|
| `Cmd+T` | New tab menu |
| `Cmd+W` | Close tab |
| `Cmd+Shift+T` | Reopen closed tab |
| `Cmd+1-9` | Go to tab by position |
| `Cmd+Tab` | Next tab |
| `Cmd+Shift+Tab` | Previous tab |

### Navigation

| Shortcut | Action |
|----------|--------|
| `Cmd+Shift+E` | Focus file explorer |
| `Cmd+Shift+G` | Focus git panel |
| `Cmd+Shift+R` | Focus run panel |
| `Cmd+Shift+F` | Global search |

### Missions & Tasks

| Shortcut | Action |
|----------|--------|
| `Cmd+M` | New mission |
| `Cmd+Shift+M` | Mission list |
| `Cmd+Enter` | Send message (in mission chat) |

### Editor

| Shortcut | Action |
|----------|--------|
| `Cmd+S` | Save |
| `Cmd+Z` | Undo |
| `Cmd+Shift+Z` | Redo |
| `Cmd+F` | Find |
| `Cmd+H` | Find and replace |
| `Cmd+G` | Go to line |
| `Cmd+D` | Add selection to next match |
| `Cmd+/` | Toggle line comment |
| `Cmd+Shift+/` | Toggle block comment |

### Git

| Shortcut | Action |
|----------|--------|
| `Cmd+Shift+G` | Focus git panel |
| `Cmd+Enter` | Commit (when in git panel) |

## Responsive Design

### Desktop (â‰¥1400px)
- Full layout: sidebar + main + optional right panel
- Bottom panel can be tall

### Laptop (1024-1399px)
- Sidebar collapses to icons by default
- Click icon to expand temporarily
- Bottom panel medium height

### Tablet (768-1023px)
- Sidebar hidden by default (slide out)
- Bottom panel as overlay
- Single main content area

### Mobile (<768px)
- Not primary target, but functional
- Tab drawer instead of tab bar
- Full-screen content areas
- Slide-out panels for everything

## Theming

### Color Scheme

**Dark Theme (Default):**
```css
--bg-primary: #1e1e2e      /* Main background */
--bg-secondary: #181825    /* Sidebar, panels */
--bg-elevated: #313244     /* Cards, dropdowns */
--text-primary: #cdd6f4    /* Main text */
--text-secondary: #a6adc8  /* Muted text */
--accent-primary: #89b4fa  /* Links, selections */
--accent-success: #a6e3a1  /* Success states */
--accent-warning: #f9e2af  /* Warnings */
--accent-error: #f38ba8    /* Errors */
--border: #45475a          /* Borders */
```

**Light Theme:**
```css
--bg-primary: #eff1f5
--bg-secondary: #e6e9ef
--bg-elevated: #ffffff
--text-primary: #4c4f69
--text-secondary: #6c6f85
/* ... etc */
```

### Status Colors

| Status | Color | Usage |
|--------|-------|-------|
| Running | Blue | Active missions, processes |
| Success | Green | Completed, passed |
| Warning | Yellow | Pending approval, attention |
| Error | Red | Failed, blocked |
| Muted | Gray | Cancelled, archived |

## State Persistence

### Local Storage
- Tab configuration (order, active)
- Panel states (sidebar width, bottom panel height)
- Theme preference
- Recent files (per project)
- Recent commands (per project)
- Search history

### Server Storage
- Projects, missions, tasks
- File contents and versions
- Git state (via actual git)
- Process history
- Agent conversations
